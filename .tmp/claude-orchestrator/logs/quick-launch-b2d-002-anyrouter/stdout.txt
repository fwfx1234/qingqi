**Quick-launch behaviors that are now real:**
- Row chips show the richer "上次成功 · 12ms / 上次失败 · exit N · 34ms / 上次超时 · … / 已停止 · … / 上次出错 · …" labels produced by `RunSummary::chip_label()`, instead of plain status strings.
- Tone of the chip still reflects status (success/failure/timeout/stopped/error) via the shared `run_status_color` mapping, so the visual semantics match the existing result sheet and don't regress from the old `RunStatus`-only map.
- When a run completes, the affected row's chip refreshes immediately via the inserted `RunSummary` (duration + exit code stay accurate without a full reload).
- Initial / reload path populates `latest_run_summaries` from `QuickLaunchService::latest_run_summaries`, so chips display duration/exit data on first paint after a query.

**Quick-launch behaviors still missing / not addressed (intentionally out of scope):**

- No timestamp on the chip — only duration; "when did this last run" is still only visible in the history sheet.
- Chip is not clickable; opening the most recent run still requires the explicit "历史" button.
- The chip palette doesn't distinguish `Failed` from `Error` (both use `color-danger`), inherited from `run_status_color`.
