### Which `system-settings` behaviors are now real

1. **macOS accessibility permission** — reads real `AXIsProcessTrusted()` status via FFI, displays "已授权"/"未授权"/"未知"
2. **Open accessibility settings** — "打开设置" button opens macOS System Settings → Privacy → Accessibility pane; re-reads permission state after
3. **Open data/config/log directories** — "打开" buttons next to each diagnostics path call `platform::shell::open_directory`, show success/failure notices
4. Theme mode switching — preserved via `ThemeStore` (was already real)
5. Plugin window retention — preserved via `SettingsStore` (was already real)
6. App index status/rescan — preserved via `AppIndexService` (was already real)
7. Diagnostics paths — preserved, shown from `AppPaths::data_dir()` (was already real)

### Which `system-settings` behaviors are still missing

1. **System follow** — listening for macOS appearance changes and auto-switching when `ThemeMode::System` is active
2. **Plugin import (directory/ZIP)** — requires plugin loading architecture; honestly marked "尚未实现"
3. **Installed plugin management** — enable/disable/uninstall plugins
4. **Clipboard/file/screen-recording permissions** — macOS has no cheap public API to check these without triggering prompts; honestly marked "尚未实现"
5. **Icon cache cleanup** — honestly marked "尚未实现"
6. **Log diagnostics** — honestly marked "尚未实现"
