## Behaviors Now Real

- **Icon cache truthfulness**: `validate_cached_icon()` decodes cached PNGs to verify they're valid images; zero-byte and corrupt files are automatically removed and treated as cache misses
- **Broken path cleanup**: `clear_broken_icon_paths()` runs on cache load and after every refresh, ensuring the UI never receives broken icon paths
- **Honest fallback**: When icon extraction fails (missing, corrupt, or unreadable), `icon_path` stays `None` and the UI renders the letter tile instead
- **Noisy failure avoidance**: Bad cache files are deleted on first encounter, so subsequent scans won't retry the same corrupted file
- **CamelCase alias expansion**: "VSCode" → "vs code", "MicrosoftWord" → "microsoft word" as searchable aliases
- **Normalized bundle-id search**: "com.apple.Safari" → "comapplesafari" added to search text; normalized queries match stripped variants
- **Existing search preserved**: Name, path, bundle-id, and original alias search behavior unchanged

## Behaviors Still Missing

- **Virtual list**: Large result sets still use fixed 40-item pagination, not virtualized scrolling
- **Icon eviction on app removal**: No periodic cleanup of orphaned icon cache files for apps that were uninstalled
- **Icon cache size limit**: No cap on total icon cache disk usage
- **Search relevance scoring**: Alias matches aren't ranked higher than name matches; all terms weighted equally
- **Adaptive icon formats**: Only PNG output; no WebP or resolution-aware caching for Retina displays
