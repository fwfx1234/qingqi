## download-manager settings/filter behaviors now real

**Filters (12 tabs, all data-driven):**
- Status filters: All, Active, Paused, Completed, Failed — each shows real count from `TaskCounts`
- Category filters: Video, Audio, Document, Archive, Image, Software, Other — each shows real count, only displayed when count > 0 or active
- Filter counts come from a single `GROUP BY status, category` query via `store.task_counts()`

**Settings controls (all service-backed, persist to SQLite):**
- Save root directory (text input)
- Max concurrent (1–16, text input)
- Speed limit KB/s (text input, 0 = unlimited)
- Timeout seconds (1–3600)
- Retry count (0–10)
- Proxy URL (auto-prepends `http://` if no scheme)
- User-Agent
- Referer
- Cookie
- Custom request headers (multi-line, `Key: Value` per line)
- Settings overlay opens from bottom bar "设置" button, Save/Cancel buttons

**Other improvements:**
- Bottom bar shows save dir, concurrency, speed limit summary
- "全部恢复" (resume all) button wired through `service.resume_all()`
- Filter chip shows count badge per tab

## download-manager behaviors still missing

- No native macOS folder picker for save root (uses text input; `FolderDialog`-style picker would need GPUI file dialog integration)
- No virtualized/paginated task list (renders all tasks; should use `Page<T>` for large lists)
- No keyboard shortcuts (space to pause/resume, delete to cancel)
- Speed limit slider (currently numeric input; QML has a slider)
- Drag-and-drop URL import
- Resume on startup for in-progress tasks (currently resets to Pending)
