## Final Summary

### Exact files changed

1. **`src/features/http_capture/model.rs`** — Added 6 convenience methods (`has_request_headers`, `has_response_headers`, `has_request_body`, `has_response_body`, `request_headers_entries`, `response_headers_entries`, `timing_summary`) + 5 new tests
2. **`src/features/http_capture/view.rs`** — Tab-based detail inspector, `hide_static` filter toggle, post-query in-memory static filtering, detail mini-bar
3. **`src/features/image_compress/view.rs`** — Fixed 2 pre-existing test compilation errors (missing `batch_total` field, moved-value borrow) that blocked `cargo test --bin qingqi`

### Which http-capture inspector behaviors are now real

- **Tab-based detail switcher** using the existing `DetailTab` enum (`请求头` / `请求体` / `响应头` / `响应体` / `计时`). Clicking a tab switches what's shown in the right detail panel.
- **Header display** parses `request_headers_json` / `response_headers_json` via `HeaderEntry::from_json()` and renders each as a `name: value` row.
- **Body display** renders `request_body` / `response_body` as text.
- **Timing display** renders `duration_ms`, request/response sizes, and protocol in a formatted summary.
- **Truthful empty states** — tabs with no data show `请求头无数据`, `请求体无数据`, etc.
- **Detail summary bar** — a 4-cell mini-bar at the top of the inspector showing status, duration, request size, response size.
- **`hide_static` filter toggle** — chip in the filter panel next to "错误" and "HTTPS". Excludes `.css`, `.js`, `.png`, `.svg`, `.woff2`, etc. from the list via in-memory post-query filtering.
- **Engine-not-wired status** preserved unchanged — status bar still shows "捕获引擎未接入 — 当前仅展示已持久化的抓包数据" when no exchanges are present.

### Which http-capture behaviors are still missing

- Live proxy capture engine (mitmproxy integration or custom proxy)
- HTTPS certificate generation, installation, and trust
- System proxy takeover / recovery
- Mobile proxy capture
- Request replay from detail panel
- cURL copy / URL copy / response body save
- Composer / send-request feature
- HAR/JSON export
- Independent `service.rs` layer — filtering/queries still go directly from view to store
- `CaptureStats` is defined in the model but never populated with real computed stats
- `CaptureMethod` enum and `ProxyState`/`CertificateStatus` enums are defined but not wired to live proxy state
