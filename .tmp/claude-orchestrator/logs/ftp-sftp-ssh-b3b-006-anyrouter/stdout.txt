### ftp-sftp-ssh-client transfer behaviors that are now real
- `cancel`: marks running/queued items as `Cancelled` with `已取消` message; idempotent; refuses to overwrite already-terminal items (verified by new `cancel_does_not_overwrite_terminal_status` + `cancel_unknown_id_is_noop` tests).
- `clear_finished`: removes only `Completed/Failed/Cancelled`, preserves `Queued/Running` (verified).
- `any_active` / `any_terminal`: honest queries against current items map.
- `status_line`: queued, running (with/without speed), completed, failed (plain + message), cancelled all rendered truthfully (existing coverage preserved).
- Transfer strip header: now shows a breakdown of **活跃 / 完成 / 失败 / 取消** chips (only non-zero categories), each tinted with its semantic color (purple/green/red/muted). Surfaces existing service state more truthfully without inferring new state. Clear-finished button still appears whenever `has_terminal()` is true.
- `transfer_counts(&[TransferItem]) -> TransferCounts` helper exposed for both view and tests; covered by three new unit tests (`transfer_counts_empty_slice`, `transfer_counts_breakdown`, `transfer_counts_only_active`).

### ftp-sftp-ssh-client behaviors still missing
- FTPS backend (only FTP + SFTP wired today).
- SSH terminal — not wired; no terminal panel rendered.
- Pause/resume of transfers; retry on failed transfers.
- Pre-existing `_workers` are unused noise — `start_upload`/`start_download` spawn their own `thread::spawn` directly. The mpsc `task_tx`/`rx` is dead infrastructure; converting transfers to true `JobProvider` workers via the worker pool is deferred (would require editing `service.rs`/`pool.rs`).
- Real status messages from `run_transfer` (e.g. `传输中`/`已完成`) are written to `item.message` but the visible card uses `item.status_line()` and only surfaces `message` for the `Failed` branch — completed/cancelled already have static text and that matches existing tests.
- Encrypted/credential storage: passwords/passphrases are still plain-text in the SQLite store.
- Recent-paths persistence on profiles.
