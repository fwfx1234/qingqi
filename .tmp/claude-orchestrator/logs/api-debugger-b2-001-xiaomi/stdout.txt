## Behaviors Now Real

- **Environment create**: "+" button creates a new env with UUID, persisted to SQLite
- **Environment duplicate**: "复制" button copies current env + variables + headers into a new UUID'd row
- **Environment delete**: "删除" button deletes current env (blocked if last one)
- **Environment edit/save**: name, base_url, variables, headers all persisted via `save_environment_fields()`
- **Tab IDs**: real UUIDs (`tab-` + 12-char hex) instead of synthetic `tab-{index}`
- **Tab persistence**: tabs saved to SQLite on switch/close, restored on startup by method+url matching
- **Tab close**: ✕ button deletes tab from SQLite via service, removes from open_tabs list

## Behaviors Still Missing

- **Tab rename** (double-click or inline edit) — tab names are auto-generated from method+url
- **Tab state body/headers/params persistence** — `persist_current_tab_state()` writes the tab row but body/headers/params fields are still stubs
- **Environment import/export** — buttons exist as `meta_text` (non-clickable labels), matching suishou's reference state
- **Collection tree interaction** — no tree click → open-tab flow yet
- **WebSocket support** — not started
- **Request scenarios UI** — scenario tabs exist in model/store but no editor UI
